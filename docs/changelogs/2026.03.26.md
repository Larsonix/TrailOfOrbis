### MapImage Encoding

| Old | New |
|-----|-----|
| `MapImage.data` (`int[]` raw ARGB) | `MapImage.palette` + `MapImage.bitsPerIndex` + `MapImage.packedIndices` |

Map images changed from raw pixel arrays to **palette-based encoding**. Unique colors go in an `int[]` palette, and each pixel becomes a bit-packed index into that palette at minimum bit depth.

```java
// Migration pattern: raw ARGB pixels -> palette-encoded MapImage
Map<Integer, Integer> colorToIndex = new LinkedHashMap<>();
for (int pixel : pixels) {
    colorToIndex.putIfAbsent(pixel, colorToIndex.size());
}

int[] palette = new int[colorToIndex.size()];
for (var entry : colorToIndex.entrySet()) {
    palette[entry.getValue()] = entry.getKey();
}

int paletteSize = palette.length;
byte bitsPerIndex = (paletteSize <= 1) ? 1
    : (byte)(32 - Integer.numberOfLeadingZeros(paletteSize - 1));

int totalBits = pixels.length * bitsPerIndex;
byte[] packedIndices = new byte[(totalBits + 7) / 8];

int bitPos = 0;
for (int pixel : pixels) {
    int index = colorToIndex.get(pixel);
    for (int b = 0; b < bitsPerIndex; b++) {
        if ((index & (1 << b)) != 0) {
            packedIndices[bitPos / 8] |= (byte)(1 << (bitPos % 8));
        }
        bitPos++;
    }
}

MapImage image = new MapImage();
image.width = width;
image.height = height;
image.palette = palette;
image.bitsPerIndex = bitsPerIndex;
image.packedIndices = packedIndices;
```

## New Events

### RemovedPlayerFromWorldEvent

**Package**: `c.h.h.server.core.event.events.player`
**Type**: IEvent (global synchronous)

Fires **after** a player is fully removed from a world. Complements `DrainPlayerFromWorldEvent` (which fires during unloading, while position data is still available).

```java
getEventRegistry().registerGlobal(EventPriority.NORMAL,
    RemovedPlayerFromWorldEvent.class, event -> {
    Holder<EntityStore> holder = event.getHolder();
    World world = event.getWorld();

    // Control leave messages
    event.setBroadcastLeaveMessage(false);
    event.setLeaveMessage(Message.raw("Custom leave text"));
});
```

**Use cases**: Final cleanup, custom leave messages, instance/realm exit handling.

### BlockReplaceEvent

**Package**: `c.h.h.server.core.modules.block`
**Type**: ECS event (requires `EntityEventSystem<ChunkStore, BlockReplaceEvent>`)

Fires when a block at a position is replaced (player building, world gen, game logic).

```java
event.getChunkRef();   // Ref<ChunkStore>
event.getSelfX();      // block X in chunk
event.getSelfY();      // block Y in chunk
event.getSelfZ();      // block Z in chunk
event.getNewEntity();  // Holder<ChunkStore> for new block
```

**Use cases**: Block change logging, build detection, instance modification tracking.

## New APIs

### Entity Effects: Native Queries

New methods on `EffectControllerComponent` -- no more maintaining your own effect tracker:

| Method | Returns | Description |
|--------|---------|-------------|
| `hasEffect(EntityEffect effect)` | `boolean` | Check by effect asset |
| `hasEffect(int effectIndex)` | `boolean` | Fast check by pre-resolved index |
| `getActiveEffects()` | `Int2ObjectMap<ActiveEntityEffect>` | All active effects (index -> effect) |
| `getActiveEffectIndexes()` | `int[]` | Just the active indexes |
| `getAllActiveEntityEffects()` | `ActiveEntityEffect[]` | Array form |

```java
EffectControllerComponent effects = store.getComponent(ref,
    EffectControllerComponent.getComponentType());

// Cache the index at startup for fast repeated checks
int burnIndex = EntityEffect.getAssetMap().getIndex("Status/Burn");

if (effects != null && effects.hasEffect(burnIndex)) {
    // entity is burning
}
```

### Emote System

Three new protocol types for player emotes:

| Type | Direction | Packet ID | Purpose |
|------|-----------|-----------|---------|
| `PlayEmote` | Client -> Server | 360 | Player requests emote by `emoteId` string |
| `UpdateEmotes` | Server -> Client | 361 | Server sends emote registry (Init/Update/Remove) |
| `ProtocolEmote` | Data structure | -- | Holds `id`, `name`, `animation`, `icon`, `isLooping` |

Emotes use `AnimationSlot.Emote` (slot 4, highest priority):

```java
// Trigger an emote server-side
AnimationUtils.playAnimation(playerRef, AnimationSlot.Emote, "emote_id", componentAccessor);
```

**Use cases**: Celebration animations, emote-gated interactions, detecting player emote usage via `PlayEmote` packet listener.

### PlaySoundEventLocalPlayer

**Packet ID**: 362 -- play sounds **only to the target player**, not broadcast to nearby players.

| Field | Type | Description |
|-------|------|-------------|
| `localSoundEventIndex` | `int` | Sound index for 2D (UI) playback |
| `worldSoundEventIndex` | `int` | Sound index for 3D (world) playback |
| `category` | `SoundCategory` | SFX, UI, Music, Ambient |
| `volumeModifier` | `float` | Volume multiplier (1.0 = normal) |
| `pitchModifier` | `float` | Pitch multiplier (1.0 = normal) |

**Use cases**: Personal audio feedback (skill activations, level-ups, quest notifications) without spamming nearby players.
